# Phase 2: Storage — Supabase Storage → S3

## What We Have Now
- Two Supabase Storage buckets:
  - **`resources`** — private, group files. Path: `{groupId}/{timestamp}_{safeName}.{ext}`
  - **`avatars`** — public, user profile pictures. Path: `{userId}/{filename}`
- Operations: upload, signed URL download, delete, public URL, list
- RLS policies enforce access per group membership / user ownership
- **An S3 stub already exists**: `src/lib/storage/awsS3Storage.ts`

## AWS Setup

### S3 Buckets
Create two S3 buckets:

#### `leader-app-resources` (private)
- Block all public access
- Enable server-side encryption (AES-256 or KMS)
- Enable versioning (optional, for recovery)
- CORS config for browser uploads:
```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["https://your-app-domain.com"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```

#### `leader-app-avatars` (public read)
- Enable public read via bucket policy:
```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "PublicRead",
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::leader-app-avatars/*"
  }]
}
```
- Optionally front with CloudFront for caching/CDN

### Access Pattern

**Uploads from client:**
- Option A: Pre-signed upload URLs generated by a Lambda → client uploads directly to S3
- Option B: Upload through API Gateway + Lambda (simpler but limited to ~6MB payload)
- Recommendation: **Option A** for resources (files can be large), Option B is fine for avatars (small images)

**Downloads:**
- Resources: Generate pre-signed GET URLs in Lambda (replaces Supabase signed URLs)
- Avatars: Direct public URL via S3 or CloudFront

### Pre-signed URL Lambda
```typescript
// Lambda: generate-upload-url
import { S3Client, PutObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

const s3 = new S3Client({ region: "us-east-1" });

export const handler = async (event) => {
  // Verify Cognito JWT (from API Gateway authorizer)
  // Check group membership (authorization)

  const { bucket, key, contentType } = JSON.parse(event.body);
  const command = new PutObjectCommand({ Bucket: bucket, Key: key, ContentType: contentType });
  const url = await getSignedUrl(s3, command, { expiresIn: 300 });

  return { statusCode: 200, body: JSON.stringify({ uploadUrl: url }) };
};
```

## Client-Side Changes

### Activate the existing S3 provider
In `src/lib/storage/index.ts`, swap the provider:

```typescript
// Before
import { SupabaseStorageProvider } from './supabaseStorage';
export const storage = new SupabaseStorageProvider();

// After
import { AwsS3StorageProvider } from './awsS3Storage';
export const storage = new AwsS3StorageProvider();
```

### Complete the S3 provider implementation
The stub in `awsS3Storage.ts` needs:
- S3 client initialization with region + credentials (use Cognito identity pool for client-side credentials)
- Or: call your pre-signed URL Lambda instead of using the S3 SDK directly from the client

## Data Migration

```bash
# Export from Supabase Storage and upload to S3
# Use Supabase CLI or direct API calls to download all files
# Then sync to S3:
aws s3 sync ./exported-resources/ s3://leader-app-resources/
aws s3 sync ./exported-avatars/ s3://leader-app-avatars/
```

## Authorization
- Supabase RLS on storage is replaced by:
  - **Uploads**: Lambda verifies group membership before issuing pre-signed URL
  - **Downloads**: Lambda verifies group membership before issuing pre-signed GET URL
  - **Avatars**: Public read, upload restricted to own `{userId}/` prefix via Lambda check
